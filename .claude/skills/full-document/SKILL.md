# Full Document

스크립트를 **리팩토링** → **확장** → **문서화**까지 한 번에 처리하는 통합 스킬입니다.

## Usage

```
/full-document <스크립트 경로>
```

## Arguments

- `path`: 처리할 스크립트 파일 경로 (필수)

## Examples

```
/full-document Assets/Scripts/Core/StationController.cs
/full-document Assets/Scripts/UI/HMIUIController.cs
```

---

## 실행 흐름

이 스킬은 3단계를 **순차적으로** 실행합니다:

### 📌 Step 1: Refactor (리팩토링)

기능을 유지하면서 코드를 단순화합니다.

**적용 규칙:**
- Early Return 패턴으로 중첩 제거
- null-conditional (`?.`, `??`) 연산자 사용
- 매직 넘버를 상수로 추출
- 미사용 코드/변수 제거
- LINQ 간결화 (과도하지 않게)

**금지:**
- public API 변경
- 기존 동작 변경

---

### 📌 Step 2: Expand Code (코드 확장)

초보자가 읽기 쉬운 정석 스타일로 포맷팅합니다.

**적용 규칙:**
- 한 줄 `if`문에 중괄호 `{ }` 추가
- 표현식 본문 `=>` → 블록 본문 `{ return ...; }`
- LINQ 체이닝 줄바꿈
- 복합 조건문 줄바꿈

**유지:**
- `?.`, `??` 연산자 (가독성 좋음)
- 간단한 람다 `x => x.Id`

**금지:**
- 로직 변경
- 변수명/메서드명 변경

---

### 📌 Step 3: Explain Code (문서화)

코드에 인라인 주석을 추가하고 설명 문서를 생성합니다.

**생성물:**
1. `[파일명]_explained.md` — 코드 구조 및 동작 설명 문서
2. 원본 파일에 인라인 주석 추가

**주석 규칙:**
- 한국어로 작성
- 코드 동작 위주의 기술적 설명
- Unity/C# API 사용 시 해당 API 역할 명시
- 이미 주석이 있는 줄은 스킵

---

## 최종 출력물

| 출력물 | 설명 |
|--------|------|
| 리팩토링된 코드 | 단순화 + 모던 문법 적용 |
| 확장된 코드 | 중괄호/줄바꿈 정석 스타일 |
| 인라인 주석 | 각 코드 줄에 한국어 설명 |
| `_explained.md` | 전체 구조 설명 문서 |

---

## 실행 지시사항

1. **파일 읽기**
   - 제공된 경로의 파일을 읽습니다
   - 파일이 없으면 사용자에게 알립니다

2. **Step 1: Refactor 실행**
   - 리팩토링 원칙에 따라 코드 단순화
   - 변경 사항 간략히 설명

3. **Step 2: Expand Code 실행**
   - 확장 규칙에 따라 포맷팅
   - 로직은 절대 변경하지 않음

4. **Step 3: Explain Code 실행**
   - `_explained.md` 문서 생성
   - 원본 파일에 인라인 주석 추가

5. **완료 보고**
   - 3단계 처리 결과 요약
   - 생성된 파일 목록 표시

---

## 주의사항

- 각 단계는 **순차적으로** 실행 (병렬 X)
- 이전 단계의 결과물 위에 다음 단계 적용
- 최종 결과는 3단계 모두 적용된 상태
- 외부 의존성/프레임워크 규칙 준수
