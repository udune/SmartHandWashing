# refactor

스크립트를 기능은 유지하면서 심플하고 읽기 쉬운 코드로 리팩토링하는 스킬입니다.

## Usage

```
/refactor <스크립트 경로>
```

## Arguments

- `path`: 리팩토링할 스크립트 파일 경로 (필수)

## Examples

```
/refactor Assets/Scripts/Core/StationController.cs
/refactor ./src/utils/helper.ts
```

## Skill Prompt

사용자가 제공한 스크립트를 심플한 코드로 리팩토링합니다.

### 리팩토링 원칙

1. **기능 100% 보존**
   - 기존 동작을 절대 변경하지 않습니다
   - public API (메서드 시그니처, 이벤트 등)를 유지합니다
   - 테스트가 있다면 모든 테스트가 통과해야 합니다

2. **코드 단순화**
   - 불필요한 중첩(nesting) 제거 → Early Return 패턴 사용
   - 중복 코드 제거 → 헬퍼 메서드 추출
   - 복잡한 조건문 단순화 → 가독성 좋은 변수명 사용
   - 사용하지 않는 코드/변수 제거

3. **가독성 향상**
   - 명확한 변수/메서드 이름 사용
   - 한 메서드는 한 가지 일만 수행 (Single Responsibility)
   - 매직 넘버를 상수로 추출
   - 적절한 공백과 그룹핑

4. **모던 문법 적용**
   - null-conditional 연산자 (?., ??)
   - 패턴 매칭 (is, switch expressions)
   - 표현식 본문 멤버 (=>)
   - LINQ 활용 (과도하지 않게)

### 실행 단계

1. **스크립트 읽기**
   - 제공된 경로의 파일을 읽습니다
   - 파일이 존재하지 않으면 사용자에게 알립니다

2. **코드 분석**
   - 현재 코드의 구조와 기능을 파악합니다
   - 리팩토링 가능한 부분을 식별합니다
   - 변경 계획을 사용자에게 간략히 설명합니다

3. **리팩토링 적용**
   - 원칙에 따라 코드를 수정합니다
   - 변경 내용을 명확히 설명합니다

4. **결과 요약**
   - 주요 변경 사항을 요약합니다
   - Before/After 비교 포인트를 제시합니다

### 주의사항

- 외부 의존성이나 프레임워크 규칙을 준수합니다
- 팀 코딩 컨벤션이 있다면 (CLAUDE.md 참조) 따릅니다
- 과도한 추상화는 피합니다 — 심플함이 목표입니다
- 주석은 "왜"를 설명할 때만 추가합니다
